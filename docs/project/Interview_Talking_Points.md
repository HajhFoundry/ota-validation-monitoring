# Interview Talking Points

## Why did you build this project?

To simulate a real automotive OTA validation ecosystem.

---

## Technologies Used

- Python
- FastAPI
- Streamlit
- MQTT
- Pytest
- JSON

---

## What problems does it solve?

- OTA readiness validation
- Campaign targeting
- Safety validation
- Cybersecurity validation
- OTA monitoring
- Regression testing

---

## Key Features

### Eligibility Validation

Checks:

- Battery
- WiFi
- Ignition
- Driving state

---

### Safety Validation

Checks:

- Vehicle driving status
- Driver distraction prevention

---

### Cybersecurity Validation

Checks:

- TLS
- Certificates
- Package signature
- Checksum

---

### OTA Monitoring

Tracks:

- OTA state
- Progress
- Campaign status

---

### Automation

JSON-driven regression testing using pytest.
